# Analysis_LLM_Clinical_Extraction
Research code for analyzing temperature sensitivity in LLM-generated clinical extraction outputs. Includes methods for JSON parsing, patient-level disagreement analysis, and evaluation of model and prompt stability across decoding temperatures.

Temperature Sensitivity in LLM Clinical Information Extraction

This repository contains analysis code for evaluating temperature-driven variability in large language model (LLM) outputs for clinical information extraction.

The analysis examines structured JSON predictions generated by LLMs from CT report narratives, focusing on how decoding temperature affects output stability, consistency, and disagreement across runs.

Because the underlying clinical narratives contain protected health information (PHI), the dataset and inference pipeline used to generate the model outputs are not included in this repository. This repo contains only the analysis code used to evaluate those outputs.

**Project Goals**

This project investigates how stochastic decoding affects LLM extraction performance by measuring:
- Output stability across decoding temperatures
- Patient-level disagreement across repeated runs
- Temperature sensitivity across models
- Prompt design effects on output consistency
- Which temperatures most often produce unstable outputs

**Analysis Overview**

The script performs several analysis steps:

1.) JSON Parsing

LLM outputs often include reasoning text or formatting inconsistencies.
The script extracts and safely parses the first JSON block from each output.

2.) Output Normalization

To enable consistent comparisons, JSON outputs are normalized by:
- sorting keys
- sorting lists of metastatic disease sites

3.) Patient-Level Disagreement

For each patient report, the analysis counts the number of unique structured outputs generated across runs.

4.) Temperature Sensitivity

For each combination of patient, model, and prompt type, the script determines whether changing temperature alters the model’s prediction.

5.) Instability Attribution

Among inconsistent cases, the analysis identifies which temperatures most frequently produce minority predictions.

**Example Output Format**

The analysis expects a dataset where each row represents one LLM run.

Example columns:
- report_index
- narrative
- model
- prompt_type
- temperature
- output

Example model output:
````
{
  "primary_disease_site": "lung",
  "metastatic_disease_status": "present",
  "metastatic_disease_sites": ["bone", "brain"],
  "metastatic_disease_stage": "stage iv"
}
````
**Visualizations**

The script produces:

1.) Temperature Instability Plot

Proportion of unstable outputs attributed to each temperature (with 95% confidence intervals).

2.) Model vs Prompt Heatmap

Percentage of temperature-sensitive outputs across model and prompt combinations.
